0x0001<<3=0x0008

一位16进制数对应4位二进制数，左移3位，就是二进制左移
0x0001 的二进制表示是：

`0x0001 = 0000 0000 0000 0001 （16位）`  
左移 3 位

`0000 0000 0000 0001 << 3`

每位向左移动 3 位，低位补 0：

`0000 0000 0000 1000`

所以，最后一位变成了8.

## 0x1000 = 4KB这个是怎么算的

0x1000 = 1 * 16^3 + 0*16^2 + 0*16^1 + 0*16^0
        = 1 * 4096 + 0 + 0 + 0
        = 4096 (十进制)


# 单位

- **数字本身只是一个数值**，它的单位要看上下文。
    
- 在内存地址或页表里：
    
    - `0x1000` 表示 **内存地址偏移**，单位是 **字节（Byte）**，不是 bit。
        
- 如果你把它当作 bit 来算，你就会错误地认为 4096 bit = 512 Byte，而操作系统里实际对应的是 **4096 字节（4 KB）**。

### 3️⃣ 总结

- 十六进制只是表示数字的一种方式，可以很方便地和二进制互相转换。
    
- **它本身没有单位**，必须结合上下文决定是 bit 还是 Byte。
    
- ==内存、页表、文件大小一般都是 **字节（Byte）为单位**==。
