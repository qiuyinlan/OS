### 为什么可以随便用 .disk？

**答：在 Linux 中，扩展名只是"便签纸"，不影响文件功能！**

```
文件 = 内容 + 名字

内容：决定文件是什么（系统通过读取内容判断）
名字：只是方便人类识别（包括扩展名）

Windows：名字很重要（系统依赖扩展名）
Linux：  内容很重要（系统不管扩展名）
```

所以你看到的各种扩展名（`.img`, `.bin`, `.disk`, `.o`, `.elf`）都只是**程序员的命名习惯**，帮助自己和别人理解文件的用途，但对系统来说都一样！


### bochsrc.disk 是什么？

**答：** 它是 Bochs 虚拟机的"配置说明书"，告诉 Bochs 怎么启动虚拟机。文件名中的 `.disk` 只是表示"从硬盘启动"。

## .c 和 .cc 的区别

### 1. 这次扩展名**不只是给人看的**，编译器也会看！

```
.c   → gcc 认为这是 C 语言代码
.cc  → g++ 认为这是 C++ 语言代码
.cpp → g++ 认为这是 C++ 语言代码
```


他们是高级语言，编译器把高级语言——汇编——bin
汇编器就是把汇编——bin

gcc就是把高级语言——汇编。


# .img

### 为什么是 .img 而不是 .bin？

**答：只是习惯！**

```
┌─────────────────────────────────────────┐
│  .img 通常表示"镜像文件"（image）        │
│  - hd60M.img（硬盘镜像）                │
│  - floppy.img（软盘镜像）               │
│                                         │
│  .bin 通常表示"二进制文件"（binary）     │
│  - kernel.bin（内核程序）               │
│  - mbr.bin（引导代码）                  │
│                                         │
│  但本质上都是二进制数据！               │
│  你可以：                               │
│  - 把 hd60M.img 改名为 hd60M.bin       │
│  - 把 kernel.bin 改名为 kernel.img     │
│                                         │
│  只要在使用时指定正确的文件名即可！      │
└─────────────────────────────────────────┘
```



## Linux vs Windows 的区别

### Windows 系统：

- **依赖扩展名**判断文件类型
- `.exe` 才能执行
- `.txt` 用记事本打开
- `.jpg` 用图片查看器打开
- **改扩展名可能导致文件无法打开**

### Linux 系统：

- **不依赖扩展名**
- 可执行权限决定能否执行（`chmod +x`）
- 文件内容决定类型
- **扩展名只是给人看的注释**
