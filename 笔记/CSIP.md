在 x86 体系结构的 CPU 中，程序计数器（Program Counter, PC）并不是一个单独的寄存器，而是一个**寄存器组合**，指的是 **段寄存器 CS** 和 **指令指针寄存器 IP**。

它们一起决定了 CPU 将要执行的下一条指令的内存地址：

- **CS（Code Segment）**：代码段寄存器，存放当前==代码段的**基址**==；
    
- **IP（Instruction Pointer）**：指令指针寄存器，存放当前代码==段内的**偏移==地址**。

实模式：物理地址 = CS × 16 + IP

CPU 在执行时，会把 `CS:IP` 合并成一个“物理地址”，将此地址送上地址总线，CPU 根据此地址便得到了指令，并将其存入到指令寄存器 IR 中。

CS 和 IP 不能直接用 mov 指令改变，因为 mov 一次只能改变一个寄存器，而跳转必须同时修改这两个寄存器。如果只改一个，会导致执行流错误。因此，CPU 设计了专门的指令（如 jmp、call、int、ret）来同时修改 CS 和 IP，从而安全地改变程序执行流。
